# RS-485 (EIA/TIA-485)

## Overview
RS-485 is a differential serial communication standard designed for multi-drop bus applications. It supports multiple masters and slaves on a single bus over long distances (up to 1200 meters) with superior noise immunity compared to RS-232. It's widely used in industrial, automation, and SCADA systems.

## Differential Signaling

```
RS-485 uses differential pairs (A & B):

Logic 1 (Space):
  A = +1.5V to +6V
  B = -1.5V to -6V
  Vab = A - B = +3V to +12V (positive)

Logic 0 (Mark):
  A = -1.5V to -6V
  B = +1.5V to +6V
  Vab = A - B = -3V to -12V (negative)

Noise Immunity:
Common mode noise affects both A and B equally
Differential receiver ignores common mode
Only responds to A - B difference

Example with 500mV noise:
Without noise:
  A = 3V, B = 0V (Vab = 3V) → 1

With 500mV noise:
  A = 3.5V, B = 0.5V (Vab = 3V) → Still 1! ✓
  
Twisted pair ensures noise couples equally to both lines
Shielded cable further reduces EMI
```

## Typical Transmission & Reception

```
Transmitter Control (RE/DE pin):

RE/DE = HIGH: Transmit mode
  - TX+ and TX- drivers ON
  - Driver outputs differential signal
  - RX+ and RX- in high-impedance

RE/DE = LOW: Receive mode
  - TX+ and TX- drivers OFF (tri-state)
  - RX+ and RX- connected to bus
  - Receives other's transmission

Bus Arbitration:

┌─ Master 1 wants to send
  RE/DE = HIGH
  TX enables drivers
  Sends data on A & B
  
┌─ Master 2 also wants to send
  RE/DE = HIGH
  Detects A-B voltage
  Waits for bus idle
  
Bus Collision Detection:
If both transmit simultaneously:
  Driver on Master1 pulls A HIGH
  Driver on Master2 pulls B HIGH
  Collision detected!
  Both back off and retry
  
Non-destructive arbitration
Lowest address priority (standard)
```

## How RS-485 Works

### Hardware Layer:

1. **Differential Output (Transmitter)**
   ```
   MAX485 Transmitter:
   When RE/DE = HIGH (TX mode)
   
   If logic 1 to send:
     TX+ → A line (pulled to +5V approximate)
     TX- → B line (pulled to GND approximate)
     Differential voltage = +5V (easily detected)
   
   If logic 0 to send:
     TX+ → B line (pulled to GND)
     TX- → A line (pulled to +5V)
     Differential voltage = -5V (easily detected)
   ```

2. **Differential Input (Receiver)**
   ```
   MAX485 Receiver:
   Compares A - B voltage
   
   If Vab > +200mV:  Output = 1 (HIGH)
   If Vab < -200mV:  Output = 0 (LOW)
   If -200mV < Vab < +200mV: Undefined (avoid!)
   
   Hysteresis prevents false triggering near threshold
   ```

### Protocol Layers:

1. **Master Polling Sequence**
   ```
   Master sends: Query to Slave 1
   RE/DE = HIGH (enable transmitter)
   Send address + command
   RE/DE = LOW (receive mode)
   Wait for response
   
   If no response after timeout:
   Continue to Slave 2
   ```

2. **Slave Response**
   ```
   Slave monitoring bus (RE/DE = LOW)
   Detects address matches
   Formulates response
   RE/DE = HIGH (wait for bus clear)
   Wait for inter-frame delay
   Send response
   RE/DE = LOW (back to receive)
   ```

3. **Collision Avoidance**
   ```
   Common practice: Master polls slaves
   Only master initiates transmission
   Slaves never transmit simultaneously
   Eliminates collision issues
   
   For multi-master:
   Each master has unique priority address
   Carrier Sense Multiple Access with Collision Detection (CSMA/CD)
   Wait for idle bus before transmitting
   Detect collisions, back off randomly
   ```

## Interview Questions & Answers

### BEGINNER LEVEL

**Q1: What does RS-485 stand for and what's its purpose?**
A: RS-485 is a differential serial communication standard (EIA/TIA-485). Its purpose:
- Multi-drop bus (multiple devices on single pair)
- Long distance transmission (up to 1200 meters)
- Higher noise immunity than RS-232 or RS-422
- Industrial and automation applications
- Full-duplex (if using 4 wires) or half-duplex (2 wires)

**Q2: What are the main differences between RS-485 and RS-232?**
A:
| Feature | RS-232 | RS-485 |
|---------|--------|---------|
| Voltage | ±12V (single-ended) | ±6V differential|
| Multi-slave | No (point-to-point) | Yes (multi-drop) |
| Distance | ~50m | ~1200m |
| Immunity | Moderate | Excellent |
| Devices | 2 (Master-Slave) | 32 typical |
| Duplex | Full | Half-duplex typical |
| Complexity | Simple | Moderate |

**Q3: What is differential signaling and why is it used?**
A: Differential signaling uses two complementary signal lines (A and B):
- Logic 1: A HIGH, B LOW (Vab = +3 to +6V)
- Logic 0: A LOW, B HIGH (Vab = -3 to -6V)

Why it's used:
- Noise couples equally to both lines
- Receiver subtracts: Vab = VA - VB (noise cancels)
- Twisting wires ensures equal coupling
- Provides immunity to ground shifts
- Can work in electrically noisy environments

**Q4: What is the role of the 120Ω terminator resistor?**
A: Terminator resistors (120Ω) placed at both bus ends:
- Match bus characteristic impedance (120Ω for twisted pair)
- Prevent signal reflections at cable ends
- Absorb transmitted energy (prevent bounces)
- Required for long cables (>10m typical)
- Missing terminators = ringing, errors

**Q5: What is the RE/DE pin used for?**
A: RE/DE (Receive Enable / Driver Enable):
- **HIGH**: Transmitter ON, driver outputs signal
- **LOW**: Receiver ON, driver tri-stated (high-Z)

Prevents bus contention (multiple drivers simultaneously).

**Q6: How many devices can RS-485 support?**
A: Standard: 32 devices maximum
- Each device loads the bus (unit load)
- Max 32 unit loads per specification
- Can exceed 32 with lower current transceivers (1/8 load)
- Practical limit depends on cable quality and noise

**Q7: What communication protocol is typically used with RS-485?**
A: Common protocols:
- **Modbus RTU**: Industrial standard, serial protocol
- **Profibus**: Factory automation
- **DMX512**: Stage lighting control
- **Custom protocols**: Master polls slaves
- **Point-to-point**: UART frames over RS-485 bus

### INTERMEDIATE LEVEL

**Q8: Explain the RS-485 termination requirements.**
A:
```
Termination Resistor Placement:

Example: Master and 3 slaves over 50m cable

          120Ω
          Pull-up
            │
      A ────┴─────────────────────────── A
      Master                           Slave3
      Slave1 Slave2
      B ──────────────────────────────── B
            │
        120Ω (Pull-down)

Bus Impedance: 120Ω twisted pair
Termination: 120Ω resistors at ends

Why termination matters:
Without terminators:
  Signal reaches end of cable
  Reflects back (100% reflection)
  Creates ringing, overshoot
  Bits corrupted

With terminators:
  Signal absorbed by 120Ω resistor
  No reflection
  Clean signal
  Reliable transmission

When needed:
- Cables > 10 meters: REQUIRED
- Cables < 10 meters: Optional but recommended
- High speed (> 500 kbps): REQUIRED
- Low speed (< 100 kbps): May work without

Calculation:
For maximum number of nodes:
Parallel resistance = 120Ω / N
Total network resistance = 120 / (N/32) = fixed 120Ω
(Designed to self-terminate)
```

**Q9: How is master-slave communication structured in RS-485?**
A:
```
Master-Slave (Polling) Architecture:

Master continuously polls slaves:

Time 1:
Master: Transmits "Slave1, send status"
  RE/DE = HIGH (TX enabled)
  Send query
Master: WAIT for response
  RE/DE = LOW (RX enabled)
  Timeout = 100ms

Slave1: Detects address
  RE/DE = LOW (listening)
  Receives command
  If match: Prepare response

Slave1: Transmits response
  RE/DE = HIGH (TX enabled)
  Send status data
  RE/DE = LOW (back to listen)

Master: Receives response
  Process data
  Continue to Slave2

Time 2:
Master: RE/DE = HIGH
  "Slave2, send status"
  
Slave2: Responds (if address matches)

Advantages:
+ No collisions (master controls bus)
+ Simple coordination
+ Deterministic timing
+ Works with any number of slaves

Disadvantages:
- Slaves cannot initiate communication
- Response time = polling interval
- Bandwidth wasted on polls getting no response
```

**Q10: Explain RS-485 half-duplex vs. full-duplex.**
A:
```
Half-Duplex (2-wire, most common):
      Master                  Slaves
      ┌──────┐                ┌──────┐
      │      │─ A/TX+──────   │A/RX+ │
      │      │─ B/TX-──────   │B/RX- │
      │      │                │      │
      └──────┘
      
Control:
Master TX ON → All slaves RX
Master RX ON → All slaves quiet (TX OFF)
Simultaneous TX impossible

Bus wire: A + B = 2 wires
Time division: Not full-duplex
Response: ~500µs turnaround per slave

Use: Industrial, Modbus, long distance


Full-Duplex (4-wire, less common):
      Master                  Slave
      ┌──────┐                ┌──────┐
      │TX+ ──┼──────────────  │RX+───┼── RX
      │TX- ──┼──────────────  │RX-───┼── circuit
      │RX+ ◄─┼──────────────  │TX+ ──┼── TX
      │RX- ◄─┼──────────────  │TX- ──┼── circuit
      └──────┘                └──────┘
      
Control:
Master can TX while listening to RX
Simultaneous 2-way possible
Echo cancellation required

Bus wires: 4 wires (TX pair + RX pair)
True full-duplex capability

Use: High-speed applications
     Lower latency needed
     More complex circuits
     
Comparision:
                Half-Duplex    Full-Duplex
Wires:          2              4
Latency:        Higher         Lower
Complexity:     Low            Medium
Common:         Yes            Less common
Cost:           Lower          Higher
Distance:       1200m          ~400m
```

**Q11: How do you handle RS-485 bus arbitration with multiple masters?**
A:
```
Multi-Master Arbitration (CSMA/CD style):

Each master has unique address:
Master1 = 0x01 (highest priority)
Master2 = 0x02
Master3 = 0x03 (lowest priority)

Transmission Algorithm:
1. Check if bus idle (monitor A-B for 5+ µs)
2. Add random backoff based on priority
   Priority devices: 1-5ms backoff
   Lower priority: 5-10ms backoff
3. Begin transmission (RE/DE = HIGH)
4. Monitor own transmission
5. Check if output matches monitored input
   If mismatch: COLLISION!
   Collision resolution:
   - Back off immediately
   - Wait random time (exponential backoff)
   - Retry transmission

Collision Example:
Master1: Starts TX at time 0
Master2: Also tries TX at time 0.1ms
  Both pull different voltages on A & B
  Glitch detected!
  Signals get mangled

Detection:
Master monitors: If(my_output ≠ bus_actual)
  Collision!
  Release bus
  Wait (random) × (retry_count²) ms
  Retry

Code Example:
bool arbited_tx(uint8_t *data, uint8_t len) {
    // Check bus idle
    for (int i = 0; i < 5; i++) {
        if (IS_BUS_ACTIVE) {
            return false;  // Bus busy
        }
        _delay_us(1);
    }
    
    // Transmit
    RE_DE_HIGH;  // Enable TX
    
    // Monitor for collision
    for(int i = 0; i < len; i++) {
        uart_send_byte(data[i]);
        
        if (uart_read_byte() != data[i]) {
            // Collision detected!
            RE_DE_LOW;
            backoff = random(1, 100) * my_address;
            _delay_ms(backoff);
            return false;  // Retry
        }
    }
    
    RE_DE_LOW;  // Back to RX
    return true;  // Success
}

Disadvantages:
+ Deterministic only for single master
- Complex with multiple masters
- Network collision overhead
- IEEE 802.3 CSMA/CD not quite applicable
```

**Q12: Design a Modbus RTU system with RS-485.**
A:
```cpp
#include <avr/io.h>
#include <stdint.h>

#define RS485_RXTX PC2  // RE/DE GPIO pin
#define MODBUS_SLAVE_ID 0x01

// Modbus RTU Frame Structure
typedef struct {
    uint8_t slave_id;
    uint8_t function;
    uint8_t data[252];  // Variable length
    uint16_t crc16;
} modbus_frame_t;

// Modbus Function Codes
#define FUNC_READ_COILS            0x01
#define FUNC_READ_DISCRETE         0x02
#define FUNC_READ_HOLDING_REG      0x03
#define FUNC_READ_INPUT_REG        0x04
#define FUNC_WRITE_SINGLE_COIL     0x05
#define FUNC_WRITE_SINGLE_REG      0x06
#define FUNC_WRITE_MULTIPLE_REGS   0x10

// CRC-16 (CCITT) calculation for Modbus
uint16_t modbus_crc16(uint8_t *data, uint8_t len) {
    uint16_t crc = 0xFFFF;
    
    for (int i = 0; i < len; i++) {
        crc ^= data[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 0x0001) {
                crc = (crc >> 1) ^ 0xA001;
            } else {
                crc >>= 1;
            }
        }
    }
    
    return crc;
}

void modbus_init() {
    // UART initialization (9600, 8N1)
    UBRR0 = 103;  // 16MHz / (16 * 9600) - 1
    UCSR0B = (1 << TXEN0) | (1 << RXEN0) | (1 << RXCIE0);
    UCSR0C = (1 << UCSZ00) | (1 << UCSZ01);
    
    // RS485 control pin
    DDRC |= (1 << RS485_RXTX);
    PORTC &= ~(1 << RS485_RXTX);  // RX mode initially
}

void modbus_uart_transmit(uint8_t *data, uint8_t len) {
    PORTC |= (1 << RS485_RXTX);  // TX mode
    _delay_ms(1);  // Stabilization time
    
    for (int i = 0; i < len; i++) {
        while (!(UCSR0A & (1 << UDRE0)));
        UDR0 = data[i];
    }
    
    // Wait for last byte to transmit
    while (!(UCSR0A & (1 << TXC0)));
    _delay_ms(1);  // Post-transmission delay
    
    PORTC &= ~(1 << RS485_RXTX);  // RX mode
}

void modbus_send_response(uint8_t func, uint8_t *resp_data, uint8_t data_len) {
    uint8_t frame[256];
    uint8_t index = 0;
    
    // Build response frame
    frame[index++] = MODBUS_SLAVE_ID;
    frame[index++] = func;
    
    // Copy response data
    for (int i = 0; i < data_len; i++) {
        frame[index++] = resp_data[i];
    }
    
    // Calculate CRC
    uint16_t crc = modbus_crc16(frame, index);
    frame[index++] = (crc & 0xFF);        // CRC-Low
    frame[index++] = ((crc >> 8) & 0xFF); // CRC-High
    
    // Transmit frame
    modbus_uart_transmit(frame, index);
}

// Function 0x03: Read Holding Registers
void modbus_read_holding_registers(uint8_t *req_data) {
    uint16_t start_addr = (req_data[0] << 8) | req_data[1];
    uint16_t num_regs = (req_data[2] << 8) | req_data[3];
    
    if (num_regs > 125) {
        // Exception: Invalid data value
        uint8_t exception = 0x83;  // 0x03 | 0x80
    }
    
    // Prepare response
    uint8_t byte_count = num_regs * 2;
    uint8_t response[251];
    uint8_t index = 0;
    
    response[index++] = byte_count;
    
    // Read registers (example: temperature sensors)
    for (int i = 0; i < num_regs; i++) {
        uint16_t reg_value = read_holding_register(start_addr + i);
        response[index++] = (reg_value >> 8) & 0xFF;
        response[index++] = reg_value & 0xFF;
    }
    
    modbus_send_response(0x03, response, index);
}

// Function 0x06: Write Single Register
void modbus_write_single_register(uint8_t *req_data) {
    uint16_t reg_addr = (req_data[0] << 8) | req_data[1];
    uint16_t reg_value = (req_data[2] << 8) | req_data[3];
    
    write_holding_register(reg_addr, reg_value);
    
    // Echo request as response
    modbus_send_response(0x06, req_data, 4);
}

// Main Modbus handler
int main() {
    modbus_init();
    sei();  // Enable interrupts
    
    while (1) {
        // Main loop can do other tasks
        // Modbus reception handled by interrupt
    }
    
    return 0;
}

// UART RX Interrupt
ISR(USART0_RX_vect) {

    if (frame_index >= 256) {
        // Buffer overflow
        frame_index = 0;
    }
```

**Q13: Calculate maximum RS-485 cable length for given baud rate.**
A:
```
Factors:
1. Baud rate
2. Cable capacitance (75-100 pF/meter typical)
3. Cable resistance (depends on gauge)
4. Termination resistance

General Formula:
Cable Length ≈ (Baud Rate / 60,000) maximum distance in meters

Examples:
9,600 bps:    Max ~200m (formula) actual ~1200m (spec)
19,200 bps:   Max ~320m (formula) actual ~600m
38,400 bps:   Max ~640m (formula) actual ~300m
115,200 bps:  Max ~1900m (formula) actual ~100m

More accurate method:
Use bit transition time and propagation delay

For differential signaling:
Max distance = Bit_Period / (2 × Propagation_Delay)

Cable propagation: ~0.66c (66% of light speed)
Propagation delay: ~1.5 ns/meter

Example (9600 bps):
Bit period = 1/9600 = 104.2 µs
Propagation delay = 1.5 ns/m

Max distance = 104.2 µs / (2 × 1.5 ns/m)
            = 104.2 µs / (3 ns/m)
            = ~34.7 km theoretical

But practical limits (reflections, noise):
- 9.6 kbps: 1200m (standard spec)
- 38.4 kbps: 400m practical
- 115.2 kbps: 100m practical
- 1 Mbps: 50m typical

Rule of thumb:
Distance × Baud Rate ≤ 120,000,000

Examples:
- 1000m × 115,200 bps = 115,200,000 (OK, ~1200m spec)
- 100m × 1,000,000 bps = 100,000,000 (OK, ~100m typical)
- 50m × 3,000,000 bps = 150,000,000 (Exceeds, reduce distance)

Improvements for longer distances:
1. Use lower baud rate
2. Use better quality cable
3. Add repeaters/boosters
4. Use fiber optic converters
5. Reduce stub lengths
6. Use proper termination
```

## Advantages & Disadvantages

### Advantages ✓
- Multi-drop support (up to 32 devices)
- Very long distances (1200 meters @ 9600 bps)
- Excellent noise immunity (differential signaling)
- Industrial standard (widely used)
- Moderately priced transceivers
- Half-duplex or full-duplex modes
- Works in electrically noisy environments
- Modbus RTU standard support

### Disadvantages ✗
- More complex than single-ended (RS-232)
- Requires differential transceiver IC
- Half-duplex introduces latency
- Requires proper termination
- Twisted pair cable required (additional cost)
- Limited bandwidth compared to modern protocols
- Single master typical (multi-master complex)
- Not suitable for very high-speed applications (>10 Mbps)

## Troubleshooting Guide

| Issue | Cause | Solution |
|-------|-------|----------|
| No communication | Missing terminator | Add 120Ω at bus ends |
| Data corruption | Noise/EMI | Use shielded cable, better grounding |
| All devices offline | Wrong control pin | Check RE/DE logic (HIGH=TX, LOW=RX) |
| Intermittent errors | Poor connections | Reseat connections, inspect cable |
| Slow response | Low baud rate | Increase baud rate if devices support |
| Bus stuck | Contention/multi-TX | Ensure only one transmitter active |
| Limited distance | Too high baud rate | Reduce baud rate or use repeater |
